## 1. One-liner

**코드뉴비("코뉴")가 학습 중 막힌 순간, 200줄 이내의 코드를 붙여 넣으면 — 픽셀 아트 인터페이스에서 숲/나무/잎 3계층으로 해설받고, 자동 태깅·아카이빙으로 학습 맥락이 누적되는 웹앱.**

---

## 2. 페르소나

### Primary persona: "코뉴" (코드뉴비)
- 개발 비전공자, AI를 통해 아이디어 수익화/일상 자동화를 시도하는 입문자
- 코드 입문 **아예 처음 ~ 1달 이내** (PRD §3.1과 일치)
- 강의·책·자기학습 병행
- 첫 사용자는 제품 오너 본인 (= 가장 디테일한 페르소나)

### 마스코트
- **앵무새 (썬코뉴어 모티프)** — IP 자산화 가능, OSMU 확장 적합
- **칭호 4단계 이모지 매핑** (누적 분석 횟수 기준): 🥚🦜 알에서 깨어난 코뉴 (0~9) → 🪶🦜 첫 비행에 성공한 코뉴 (10~29) → 🌲🦜 숲을 아는 코뉴 (30~99) → 🎩🦜 엣헴 코뉴 (100+). Pre-MVP는 이모지 조합으로 시각화, P1에서 캐릭터 자산 진화로 대체. ※ 칭호(누적 횟수 4단계)와 학습자 레벨(자가진단 1·2·3)은 별개 체계. → PRD FR-GAME-003

## 3. 트리거 (Activation moment)

> 강의/책/혼자 공부 중 "이게 뭐지? 왜 이렇게 쓰지? 어디서 쓰는 거지?" 하는 의구심이 떠오를 때

- 환경: 주로 PC (모바일 지원은 v1.x로 후순위)
- 시간대: 아무 때나, 단 학습자가 능동적으로 켜는 도구

## 4. 차별점 (Why not ChatGPT / Cursor / Copilot)

| 측면 | ChatGPT 등 일반 AI | Cursor/Copilot/Codex | **Code Decoder** |
|---|---|---|---|
| 응답 단위 | 단발성 | IDE 통합 단발성 | **3계층 구조화** |
| 맥락 유지 | 채팅 종료 시 휘발 | 세션 단위 | **로드맵 기반 누적** |
| 사용자 수준 적응 | 약함 | 약함 | **누적 학습 데이터 기반** |
| 학습 이력 자산화 | 없음 | 없음 | **자동 태깅 + 아카이브** |
| 위치 자각 (= 지금 어디?) | 없음 | 없음 | **계층 시각화 (폴더 트리)** |

> **참조**: 경쟁사 상세 분석 → `00-competitive-analysis.md`

### 핵심 차별화 3종 묶음 (Triple Differentiator)
1. **로드맵 기반 길찾기**: 프론트엔드/백엔드/DB 전체 로드맵에서 현재 위치 파악
2. **계층 시각화**: 숲/나무/잎 + 폴더 트리 병행 표현
3. **누적 맥락**: 자동 태깅 + 아카이빙으로 학습 흔적 보존

## 5. 입력 / 출력 스펙

### 입력
- **단위**: 텍스트 코드 스니펫
- **최대 길이**: 200줄 권장 (Claude.md 컨벤션 차용)
- **언어**:
  - MVP 1순위: **Python**
  - v1.1 추가: **JavaScript**
- **입력 전처리**: 분석 호출 전 주석·연속 공백·빈 줄 자동 제거. 단 **라인 번호는 원본 기준 보존** (사용자가 본 라인 번호 그대로). → PRD FR-ANALYSIS-006

### 출력
| 레이어 | 내용 | 토큰 추정 (한국어 부풀음 고려) |
|---|---|---|
| **Forest (숲)** | 전체 코드의 역할·맥락 요약 | ~600 |
| **Tree (나무)** | 핵심 블록(함수/섹션) 단위 분해 | ~2,000 |
| **Leaf (잎)** | **Tier 분리** — 모든 실질 라인 짧은 해설 1줄 + 핵심 5개 깊은 해설 (추가 Leaf는 클릭 시 확장·5:1 차감) | PRD §6.5 참조 |
| **Tags** | 자동 추출 키워드 | ~250 |
| **Key Concepts** | 학습용 개념어 정리 (사이드 카드) | ~150 |

> ⚠️ Output max_tokens = **8,000** 강제 제한 (PRD HC-2 · NFR-2). ※ v2 표기 6,000 → 8,000 상향 — PRD §6.5 정밀 토큰 예산 검산 결과 (실측 추정 ~7,550).

## 6. 학습자 수준 적응 시스템

### 온보딩 레벨 선택 (4지선다 + 미리보기)

```
[코뉴 캐릭터 말풍선]
"눈높이에 맞춘 해설을 위해 골라줘! 🦜
 (모르겠으면 4번도 OK!)"

1) python? 뱀이라는 풍문은 들어봄
2) Hello world 정도는 띄워봄
3) 저장한 py파일이 꽤 많이 늘었음!
4) 코뉴, 니가 알아서 좀 골라줘라! (자동 추천)

[ 🔍 각 레벨 설명 예시 미리보기 ] (옵션)
```

### 레벨 정의 (Level 1·2·3)

> 메뉴 4번(자동 추천)은 레벨값이 아님 — 시스템이 1·2·3 중 하나 할당. 칭호(누적 횟수 4단계)와도 별개 체계.

| 항목 | Level 1 | Level 2 | Level 3 |
|---|---|---|---|
| 자가진단 (메뉴) | "python? 뱀이라는 풍문은 들어봄" | "Hello world 정도는 띄워봄" | "저장한 py파일이 꽤 많이 늘었음!" |
| 대상 | 절대 초심자 · 코딩 경험 거의 0 | 입문 직후 · 기초만 손댐 | 경험 있는 학습자 · 반복 경험 보유 |
| 용어 풀이 | 모든 기술 용어에 괄호 풀이 (변수·함수도) | 핵심·중급 용어만 풀이 | 풀이 없이 사용, 고급 개념만 짚음 |
| Leaf 상세도 | 라인별 매우 상세 | 핵심 라인만 상세 | 비표준·까다로운 라인만 |
| 비유 사용 | 일상 비유 적극 (변수=상자) | 가끔 사용 | 거의 안 함 |
| 출력 길이 | 길게 | 표준 | 압축 |
| 톤 | 격려·안심 ("누구나 헷갈려") | 학습 파트너 | 담백한 동료 |

### 레벨 갱신
- 명시적: 설정 페이지에서 언제든 변경
- 자동: 30일마다 사용 패턴 분석 → "한 단계 올려볼래요?" 제안
- 4번 선택 시: 레벨 1 디폴트 + 첫 5회 분석 후 자동 재추천

### 온보딩 이탈 방지
- 레벨 선택은 **건너뛰기 가능** (= 레벨 1 자동 적용)
- 30초 안에 첫 분석 결과 보여주기가 절대 원칙

## 7. 핵심 사용자 흐름 (Top use cases)

### UC-1: 즉시 해설
> 강의 중 모르는 코드 등장 → Code Decoder 열기 → 붙여넣기 → 3계층 해설 즉시 보기

> "rag에서 window 관련 기능이(함수가) 있었는데..."
> → 태그 + 자연어 검색으로 과거 분석 이력에서 회수

### UC-3: 학습 진척 시각화
> 분석 누적 → 어떤 개념을 얼마나 봤는지 자동 집계 → 다음에 봐야 할 학습 영역 추천 (v1.x)

## 8. 자동 태깅 / 아카이빙 정책

| 항목 | 저장 정책 |

| 원본 코드 | 항상 저장 |
| Forest 요약 | 항상 저장 |

## 10. 게이미피케이션 (핵심 원칙: 0원 자원만 보상)

### 보상 가능 항목 (한계비용 ₩0)
| 행동 | 보상 |
|---|---|
| 코드 분석 1회 완료 | 🐛 캐러필러 +1 |
| 매일 1회 분석 (연속) | 🔥 Streak +1 |
| Streak 3·7·14·21일 / 30일 풀 | 마일스톤 랜덤 보상 / 풀 Streak 보상 |
| 누적 분석 10·30·100건 도달 | 단계별 칭호 진급 + 캐러필러 보너스 (칭호 이모지 4단계는 §2 마스코트 참조) |
| 새 개념(Key Concept) 발견 | 도감 슬롯 채움 *(캐러필러 보너스 없음)* |
| 도감 1·10·30개 마일스톤 | 뱃지 |
| 캐러필러 사용 | 캐릭터 코스튬 / 색상 변경 |

### 보상 금지 항목 (한계비용 발생)
- ❌ 비싼 모델(상위 모델 등) 분석 회수 보너스
- ❌ 무료 사용자에게 유료 모델 사용권
- ⚠️ **단일 명시적 예외 — 출석왕 보상권(Attendance Reward)**: 위 "유료 모델 사용권 금지" 원칙에 대한 *유일한 통제된 예외*. 전월 25일 이상 출석한 무료 사용자에게 월 1회 GPT-5 mini 5회권을 지급한다. 한계비용 약 ₩55/명/월(5회 × ₩11)로 자연 손익분기선(₩400/월/명)보다 충분히 작아 0원 원칙의 통제된 예외로 허용. Open Beta+ 단계에만 활성. → PRD HC-7 · FR-GAME-013.

### 캐러필러 정책
- **단일 통화**: XP를 폐지하고 캐러필러 단일 통화로 통합 (이중 통화 → 단일 통화). 누적 분석 횟수는 칭호 트리거용 별도 카운터로만 유지
- **MVP**: 게이미피케이션 보상 전용 (코스튬·뱃지·도감)
- **수익화 단계 진입 후**: 유료 부스터 결제 추가 (₩1,000 = 상위 모델 5회 분석 수준, 마진 약 80%). ※ 본 제품 모델군은 OpenAI GPT 계열 (PRD HC-5) — v2 표기 "Sonnet"은 v1 잔재로 일반 표현으로 정정

### 방패 시스템 (Streak Shield)
- 캐러필러 **20마리 누적 → 방패 1개 자동 변환** (캐러필러 -20, 방패 보유 0개일 때만)
- Streak 끊김 감지 시 → **자동 발동**하여 Streak 1회 방어 (방패 -1)
- 보유 한도 1개. 자동 변환 ON/OFF는 사용자 설정 가능 (디폴트 ON)
- → PRD FR-GAME-005 · FR-SETTINGS-007

### Streak 마일스톤 랜덤 보상
- Streak **3·7·14·21일** 도달 시 → 랜덤 보상, **30일 풀 Streak** → 별도 보상
- ADHD 페르소나의 도파민 회로를 겨냥한 가변 보상(variable reward) 설계
- → PRD §5.6 FR-GAME 본체

### 도감(Key Concept) 보상 정책
- 새 개념 발견 → 도감 슬롯 채움 / 1·10·30개 마일스톤 → 뱃지
- **캐러필러 보너스는 지급하지 않음** — 도감 진척은 시각 슬롯·뱃지 등 0원 자산으로만 보상
- 도감은 **개방형 누적** (고정 정원·상한 없음 — TRD §16.0 안건③)
- → PRD FR-GAME-007 · FR-GAME-008

### 풀사용자 보상 (Streak 기반) — "출석왕 보상권"
- 트리거: **전월 25일 이상 출석** (매월 1일 자정 KST 배치로 집계)
- 보상: **무료 사용자 → GPT-5 mini 5회권** (월 1회 발급 · 유효기간 다음달 말일 · 일반 분석 한도와 별도 카운터로 차감) / **유료 사용자 → 부스터 캐러필러**
- 활성 시점: **Open Beta+** (P2). Pre-MVP·Closed Beta에서는 비활성.
- → PRD FR-GAME-013 · FR-ANALYSIS-015 · FR-BILLING-010 / HC-7 예외 조항. ※ 무료 사용자 보상은 v1 Discovery의 "다음달 7일 유료 체험권"에서 변경됨 — PRD가 SSoT.

### Inspiration
- **찰리와 걷기** 만보기 앱: 누적 행동 → 캐릭터 성장 → 도파민 보상

## 13. 비기능 요구사항 (NFR) — 비용 절감 5룰 ★

PRD에서 NFR 섹션에 그대로 반영.

| # | 룰 | 효과 |
|---|---|---|
| **NFR-1** | 시스템 프롬프트 prompt caching 의무 적용 | 25-30% 비용 절감 |
| **NFR-2** | Output max_tokens = 8,000 강제 제한 | 한국어 부풀음 방지 (PRD §6.5 검산 반영) |
| **NFR-3** | 동일 코드 1주일 캐시 (해시 기반) | 재분석 비용 0 |
| **NFR-4** | 추가 Leaf(6번째+)는 lazy 생성·핀 시 저장 (핵심 5개는 분석 시 통합 생성) | 미열람 데이터 DB 비대화 방지 |
| **NFR-5** | 월별 사용자별 비용 모니터링 대시보드 | 이상치 조기 탐지 |

## 16. 위험 명시

### Risk-1: 한국어 토큰 부풀음 (~1.5배)
- **완화책**: NFR-2 (output max_tokens 강제), NFR-4 (Leaf lazy)

### Risk-2: 풀사용자 적자 (-₩299/월/인)
- **완화책**: 평균/라이트 사용자가 흑자로 보전, 부스터 결제 매출

### Risk-3: 사용자 abuse (재분석 남용)
- **완화책**: NFR-3 (1주일 코드 캐시)

### Risk-4: API 가격 인상
- **완화책**: 매분기 모델 가격 모니터링, 필요 시 모델 교체 또는 가격 조정
